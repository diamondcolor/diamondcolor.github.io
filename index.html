二、HTTP协议

  1. put和post的区别
    POST 方法用来传输实体的主体，PUT方法用来传输文件，自身不带验证机制。 这两个方法看起来都是讲一个资源附加到服务器端的请求，但其实是不一样的。
    一些狭窄的意见认为，POST方法用来创建资源，而PUT方法则用来更新资源。这个说法本身没有问题，但是并没有从根本上解释了二者的区别。
    事实上，它们最根本的区别就是：POST方法不是幂等的，而PUT方法则有幂等性。那这又衍生出一个问题，什么是幂等？

    幂等（idempotent、idempotence）是一个抽象代数的概念。在计算机中，可以这么理解，一个幂等操作的特点就是其任意多次执行所产生的影响均与依次一次
    执行的影响相同。

    POST在请求的时候，服务器会每次都创建一个文件，但是在PUT方法的时候只是简单地更新，而不是去重新创建。因此PUT是幂等的。

    举一个简单的例子，假如有一个博客系统提供一个Web API，模式是这样http://superblogging/blogs/post/{blog-name}，很简单，
    将{blog-name}替换为我们的blog名字，往这个URI发送一个HTTP PUT或者POST请求，HTTP的body部分就是博文，这是一个很简单的REST API例子。
    我们应该用PUT方法还是POST方法？取决于这个REST服务的行为是否是idempotent的，假如我们发送两个http://superblogging/blogs/post/Sample请求，
    服务器端是什么样的行为？如果产生了两个博客帖子，那就说明这个服务不是idempotent的，因为多次使用产生了副作用了嘛；如果后一个请求把第一个请求覆盖掉了，
    那这个服务就是idempotent的。前一种情况，应该使用POST方法，后一种情况，应该使用PUT方法。

在ARM 上部署Angular2 application

On build system:

Firstly install nodejs which is prerequisites for Angular CLI.
1) Download nodejs installer(Linux Binaries: node-v8.10.0-linux-x64.tar).
2) Put it into /home/lisa.
3) Tar nodejs installer:
$ tar –xvf node-v8.10.0-linux-x64.tar
4) Rename folder name:
$ mv node-v8.10.0-linux-x64.tar nodejs
5) Create soft link to config global path:
$ ln –s /home/lisa/nodejs/bin/node /usr/local/bin/node
$ ln –s /home/lisa/nodejs/bin/npm /usr/local/bin/npm
6) Check if configuration is ok
$ node –v
$ npm –v

Secondly install typescript which is prerequisites for Angular CLI.
7) $ npm intall –g typescript

Thirdly install Angular CLI.
8) $ npm intall –g @angular/cli
9) $ ng v
Until this step, Angular CLI should be installed successfully.

Fourthly build application.
10) Copy demo application to /home/lisa/myapp.
11) Cd myapp, build application.
$ ng build –prod
Then dist will be added under myapp folder. In folder dist, there are css, js, html files.

On ARM system:
12) Copy above dist to one folder, like /home/lisa/.
13) Modify nginx.conf, Set root redirect to “/home/lisa/dist”.
14) Start nginx. Ok.
You can open browser and access http://ip. Demo application is running.

开始Angular2

一、开发环境

Angular CLI: 命令行接口工具，能够创建工程，增加文件，执行一系列开发任务：测试，编译，部署等。

第一步： 建立开发环境
首先安装Node.js 和npm，然后安装Angular CLI.
npm install -g @angular/cli

第二步： 创建新工程
Ng new my-app

第三步：发布
Cd my-app
Ng server –open //ng server 命令发布服务器，查看文件，重新编译…

用http://localhost:4200/可以在浏览器打开你的app

二、生产环境
A.最简单的部署：build, copy to web server
1.Ng build
2.拷贝dist/所有的文件到服务器上的目录
3.如果要把文件到服务器上的子目录，需要加上build flag, –base-href
比如/my/app/index.html， 要用ng build –base-href=/my/app
4.配置服务器

B.生产环境的优化部署
1.Ng build –prod
–prod: 1) 预编译（AOT） 2) 生产模式 3）捆绑 4）最小化 5）丑化 6）除去无用代码

2.拷贝dist/所有的文件到服务器上的目录
3.Config nginx.conf
